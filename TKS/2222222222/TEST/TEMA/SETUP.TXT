1. Советы по быстрой и правильной установке
===========================================
Отключите все установленное антивирусное программное обеспечение. 
Если эти служебные программы будут выполняться во время установки, 
возможно зависание компьютера.
 
ПРИМЕЧАНИЕ. В некоторых компьютерах антивирусные средства встроены 
в BIOS. Если не отключить эти средства в BIOS/CMOS, при установке 
возможно появление сообщений об изменениях в системе типа 
"virus-like activity" или "Master Boot Record". 
Для успешной установки следует разрешить внести необходимые изменения. 
Более подробные сведения можно найти в документации по антивирусным 
программам.

Запустите программу ScanDisk для обнаружения и устранения возможных 
ошибок на жестком диске.

Закройте все выполняющиеся программы. Следует также отключить 
заставки, средства автоматического управления питанием и другие 
программы, которые могут помешать нормальной установке. Дополнительные 
сведения содержатся в разделе ""Чистая" загрузка системы".


2. Общие вопросы установки
==========================
При наличии установленного видеоадаптера Number Nine Imagine 128 или 
STB Velocity 128 3D AGP (Nvidia Riva 128) следует запустить программу 
установки из MS-DOS или сменить видеорежим на VGA.

Обновление и полная установка второго выпуска Windows 98
--------------------------------------------------------

При наличии версии обновления Windows 98 программа установки выполнит 
поиск в компьютере программного продукта, подлежащего обновлению. Если 
программе установки не удастся найти предыдущую версию Windows, она 
предложит вставить носитель с предыдущей версией для проверки 
соответствия. 

Дисковое пространство, необходимое для установки Windows 98 
-----------------------------------------------------------

Так как объем дискового пространства, необходимый для установки 
Windows 98, зависит от многих факторов, следующие сведения относятся 
к обычным типам установки Windows 98.

Обычное обновление Windows 95: требуется примерно 
  205 Мбайт свободного места на диске, однако в зависимости от 
  конфигурации компьютера может потребоваться до 315 Мбайт.
Полная установка Windows 98 на диск с файловой системой FAT16: 
требуется 
  260 Мбайт свободного места, эта величина может изменяться от 
  210 до 400 Мбайт в зависимости от конфигурации компьютера и 
  устанавливаемых компонентов.
Полная установка Windows 98 на диск с файловой системой FAT32: 
требуется 
  210 Мбайт свободного места, эта величина может изменяться от 
  190 до 305 Мбайт в зависимости от конфигурации компьютера и 
  устанавливаемых компонентов.

Также при установке Windows 98 не на диск C для создания системных 
файлов и журнала установки потребуется до 25 Мбайт свободного места 
на диске C.

Удаление. Если перед обновлением требуется создать резервную копию 
Windows 95, установите флажок "Сохранить системные файлы". Это 
позволит отменить установку второго выпуска Windows 98 в случае 
возникновения неполадок. Однако этого нельзя сделать в следующих 
случаях:

* Текущая версия Windows установлена на сжатом диске.
* Установка была выполнена в другой каталог или не как обновление 
  предыдущей версии.
* Используется версия MS-DOS меньше 5.0.

СОЗДАЙТЕ НОВЫЙ СИСТЕМНЫЙ ДИСК! В связи с изменениями в ядрах реального 
и защищенного режимов для поддержки файловой системы FAT32 системный 
диск второго выпуска Windows 98 не совместим с более ранними версиями 
Windows. Поэтому при первой установке второго выпуска Windows 98 
обязательно создайте новый системный диск, ДАЖЕ ЕСЛИ ИСПОЛЬЗОВАНИЕ 
ФАЙЛОВОЙ СИСТЕМЫ FAT32 НЕ ПЛАНИРУЕТСЯ.

Диспетчер программ Windows 3.x.
-------------------------------
В Windows 98 прекращена поддержка диспетчера программ. Диспетчер 
программ (Progman.exe) оставлен в системе в целях совместимости, но 
он НЕ содержит никаких групп. В дополнение к этому, при обновлении 
Windows 95 существующие файлы .grp будут удалены. Это может повлечь 
некоторые неполадки при установке Windows 98. При обновлении 
Windows 3.x имеющиеся файлы .grp останутся в системе, а диспетчер 
программ по-прежнему сможет работать. Если вы хотите использовать 
диспетчер программ, следует создать резервные копии файлов progman.ini 
и *.grp перед установкой Windows 98 в режиме обновления.


3. Установка второго выпуска Windows 98 из MS-DOS
=================================================
При установке второго выпуска Windows 98 на новый жесткий диск, а 
также в случае неполадок, связанных с установкой системы из предыдущей 
версии Windows, запустите программу установки из MS-DOS. Несмотря на 
то, что установка из MS-DOS обычно является самой медленной, она часто 
является и самой безопасной, ее следует использовать, если другие типы 
установки завершаются неудачно.

Клавиши, используемые при загрузке MS-DOS
-----------------------------------------
Существует несколько способов безопасной загрузки компьютера в режиме 
MS-DOS. Наиболее простым является использование следующих клавиш:

* Microsoft Windows 98, второй выпуск
Удерживайте нажатой клавишу CTRL при загрузке компьютера. Это нужно 
для выхода в меню загрузки Windows 98 (клавиша F8 также действует, 
но при загрузке не выдается сообщение "Starting Windows 98", поэтому 
сложно определить, когда следует нажимать эту клавишу).

* Windows 95
После появления сообщения "Starting Windows 95" нажмите клавишу F8. 
Это требуется для выхода в меню загрузки Windows 95.

* MS-DOS 6.x
После появления сообщения "Starting MS-DOS" нажмите клавишу F8. 
Далее можно будет вручную загрузить или пропустить загрузку драйверов 
и системных файлов. 

* Драйверы устройств чтения компакт-дисков реального режима
Для доступа к компакт-диску второго выпуска Windows 98 необходима 
загрузка драйверов устройства чтения компакт-дисков реального режима. 
Если программа установки Windows 98 уже запускалась и был создан 
системный диск, можно воспользоваться находящимися на нем драйверами. 
При отсутствии системного диска следует запустить программу установки, 
поставлявшуюся с устройством чтения компакт-дисков.

После получения доступа к устройству чтения компакт-дисков вставьте 
компакт-диск второго выпуска Windows 98 и введите: SETUP. Запустится 
программа установки.

Изменение файлов Config.sys и Autoexec.bat
------------------------------------------

Файлы Config.sys и Autoexec.bat отвечают за загрузку программ и 
драйверов устройств при запуске компьютера (например, задание в файле 
Autoexec.bat ссылки на программу поиска вирусов вызовет автоматический 
запуск этой программы). Программа установки второго выпуска Windows 98 
с некоторыми программами и устройствами работает неправильно. Для 
удаления или отключения программы или устройства может потребоваться 
изменение файлов Config.sys и/или Autoexec.bat.

Чтобы изменить файлы Config.sys и Autoexec.bat:

1. В Windows 3.1 или 3.11 в меню "Файл" выберите команду "Выполнить", 
   введите "Sysedit" и нажмите клавишу "ENTER". В Windows 95 нажмите 
   кнопку "Пуск", выберите команду "Выполнить", введите "Sysedit" и 
   нажмите клавишу "ENTER".

2. В диалоговом окне "Config.sys" или "Autoexec.bat" введите "REM" в 
   начале строки, которая не должна выполняться.

3. Сохраните измененные файлы и перезагрузите компьютер.

Советы по установке драйверов устройств чтения 
компакт-дисков реального режима
----------------------------------------------

Если текущей операционной системой является Windows 95:
Если текущей операционной системой является Windows 95, драйвера 
устройства чтения компакт-дисков уже могут быть загружены. Если 
возможна загрузка в режиме MS-DOS и получение доступа к устройству 
чтения компакт-дисков, выполните следующие действия:

* Перезапустите компьютер и после появления сообщения 
  "Starting Windows 95" нажмите клавишу F8.
* Выберите режим загрузки "Command Prompt Only."
* В командной строке на диске C:\ введите: DosStart.bat.

Это обеспечит доступ к устройству чтения компакт-дисков.

Если доступ к устройству чтения компакт-дисков прекратился в процессе 
установки:
Если доступ к устройству чтения компакт-дисков во время установки 
программного обеспечения второго выпуска Windows 98 прекратился, 
выполните следующие действия:

* Перезапустите компьютер, после появления сообщения 
  "Starting Windows 95" нажмите клавишу F8 и выберите режим загрузки 
  "Command Prompt Only". При использовании MS-DOS загрузите компьютер 
  в режиме командной строки.
* Начните редактирование файла Autoexec.bat, введя команду: 
  Edit Autoexec.bat
* Удалите фрагмент "Rem by Windows 98 Setup" в начале строки, 
  ссылающейся на файл Mscdex.exe.
* Выйдите их программы Edit, нажав клавиши ALT-F-X и сохранив 
  измененный файл.
* Перезагрузите компьютер. Программа установки сама может продолжить 
  работу. Если этого не произойдет, запустите ее снова и выберите 
  режим безопасного возобновления установки.

Установка второго выпуска Windows 98 с жесткого диска
------------------------------------------------------

Скопировав установочные файлы на жесткий диск и выполнив установку с 
жесткого диска, можно предотвратить все возможный проблемы, связанные 
с копированием файлов и ошибками дискового ввода/вывода. При таком 
типе установки также можно выгрузить из памяти драйверы устройства 
чтения компакт-дисков для освобождения основной памяти и 
предотвращения возникновения ошибок в нижней памяти. Для локального 
копирования установочных файлов:

В Windows 95:
* Освободите дополнительно 120 Мбайт дискового пространства кроме 
  того, которое требуется для установки. Обычно для установки 
  Windows 98 в режиме обновления Windows 95 требуется 195 Мбайт.
* Создайте временную папку "W98Flat" для расположения в ней 
  установочных файлов.
* Скопируйте содержимое папки Win98 с компакт-диска второго выпуска 
  Windows 98 в созданную временную папку. Также можно скопировать 
  подкаталоги папки Win98, но при дефиците места на жестком диске это 
  делать необязательно.
* Выполните перезагрузку. После сообщения "Starting Windows 95" 
  нажмите клавишу F8 и выберите режим защиты от сбоев 
  "Safe Mode Command Prompt Only".
* Перейдите во временную папку, содержащую установочные файлы второго 
  выпуска Windows 98, и введите: SETUP.

В MS-DOS:
* Убедитесь, что устройство чтения компакт-дисков доступно. 
  Дополнительные сведения об этом приведены выше.
* Освободите дополнительно 120 Мбайт дискового пространства кроме 
  того, которое требуется для установки. Обычно для установки 
  Windows 98 в режиме обновления Windows 95 требуется 195 Мбайт.
* На диске, имеющим достаточно свободного места для хранения 
  установочных файлов, создайте временную папку "W98Flat". Для 
  создания временного каталога перейдите на выбранный диск и введите: 
  MD W98Flat.
* Перейдите в каталог Win98 на компакт-диске второго выпуска 
  Windows 98.
* Скопируйте установочные файлы второго выпуска Windows 98 во 
  временный каталог с помощью команды: Copy *.* <drive letter>\W98Flat.
* После того, как все файлы будут скопированы, перезапустите компьютер 
  и произведите "чистую" загрузку, не выполняя загрузочные файлы. 
  Дополнительные сведения представлены ниже в соответствующем разделе.
* Перейдите во временный каталог, куда были скопированы установочные 
  файлы, и запустите программу установки, воспользовавшись командой 
  SETUP.


4. "Чистая" загрузка системы
============================
Драйверы устройств, служебные и другие программы сторонних 
производителей могут помешать успешной установке. Выполнение "чистой" 
загрузки поможет решить большинство потенциальных проблем. "Чистую" 
загрузку можно выполнить:

Загрузив компьютер с дискеты:
* Загрузите компьютер с системного диска второго выпуска Windows 98. 
  Это позволит выбрать варианты загрузки: с драйверами устройства 
  чтения компакт-дисков или без них, и является "чистой" средой для 
  запуска программы установки.
* Загрузите компьютер с системного диска Windows 95 или MS-DOS. Это 
  не обеспечит доступ к устройствам чтения компакт-дисков, но поможет 
  в том случае, если копирование установочных файлов будет выполнено, 
  как описано выше.

Загрузив Windows 95 в режиме защиты от сбоев 
(Safe Mode Command Prompt Only):
* Перезагрузите компьютер и после появления сообщения 
  "Starting Windows 95" нажмите клавишу F8.
* Выберите режим загрузки "Safe Mode Command Prompt Only". Это также 
  не обеспечит доступ к устройствам чтения компакт-дисков, но поможет 
  в случае если копирование установочных файлов будет выполнено, как 
  это описано выше.

Путем пошаговой загрузки Windows 98 второго выпуска:
Если некоторые драйверы требуется загрузить вручную, выполните 
следующие действия:
* Перезагрузите компьютер и после появления сообщения 
  "Starting Windows 95" нажмите клавишу F8.
* Выберите режим загрузки "Step by Step".
* Затем отвечайте "YES" только на запросы загрузки необходимых 
  драйверов. В большинстве случаев следует соглашаться с загрузкой 
  драйвера Himem.sys.

"Чистая" загрузка Windows 95/MS-DOS с освобождением памяти:
Для увеличения объема доступной памяти внесите в файл Config.sys 
следующие изменения. Эти изменения также можно внести на системный 
диск. 
ПРИМЕЧАНИЕ. При использовании этого файла будут загружены лишь 
указанные драйверы.

Device=Himem.sys
Device=EMM386.exe noems
Dos=high,umb
Device=drvspace.sys /move 
	(необязательная строка - она требуется только 
	при использовании программы сжатия диска DriveSpace)

Создание системного диска второго выпуска Windows 98:
Если программа установки второго выпуска Windows 98 аварийно 
завершилась после копирования на жесткий диск большинства файлов, 
системный диск можно создать, запустив служебную программу 
bootdisk.bat.
* Загрузите компьютер в режиме командной строки MS-DOS.
* Перейдите в каталог Windows\command.
* Запустите программу Bootdisk.bat, которая предложит создать 
  системный диск.

На этот диск будут записаны драйверы устройств чтения компакт-дисков 
общего типа, которые можно будет использовать при следующем запуске 
программы установки.


5. Антивирусные программы
=========================
Убедитесь, что при установке второго выпуска Windows 98 не выполняются 
никакие антивирусные программы. Если такая программа является 
резидентной, удалите из файлов Autoexec.bat, Config.sys и Win.ini 
все ссылки на нее.

Если в BIOS встроено средство защиты от вирусов, отключите его перед 
запуском программы установки. Для этого следует воспользоваться 
программой настройки CMOS для имеющейся BIOS. Более подробные сведения 
можно найти в документации по компьютеру.

Ниже приведены замечания о некоторых антивирусных программах.

Встроенное в CMOS/BIOS средство защиты от вирусов:
В некоторых компьютерах антивирусные средства встроены в BIOS. Если не 
отключить эти средства, при установке возможно появление сообщений об 
изменениях в системе типа "Virus-like Activity" или 
"Master Boot Record Changed". Следует разрешить внести необходимые 
изменения. В противном случае возможно повреждение загрузочных файлов 
и невозможность дальнейших загрузок компьютера.

Norton AntiVirus:
Если установлена программа Norton AntiVirus, в конце начальной 
процедуры копирования файлов возможно появление следующего 
предупреждения:

	Application Wininst0.400\Suwin.exe is 
	attempting to update the Master Boot Record

Для успешного завершения установки следует продолжить выполнение 
(пункт "Continue", клавиша "C"). Если не разрешить программе установки 
внести эти изменения, она может зависнуть.

Dr. Solomon's AntiVirus:
Если в процессе установки выполняется служебная программа 
Dr. Solomon's AntiVirus, при попытке создания системного диска может 
возникнуть сообщение о неустранимой ошибке в модуле Ios.vxd на синем 
фоне. Когда индикатор создания системного диска достигнет отметки 20%, 
нажмите кнопку "Отмена". Это позволит продолжить установку. Для 
решения этой проблемы попробуйте найти на веб-узле программного 
обеспечения Dr. Solomon's AntiVirus обновление.


6. Поиск и исправление ошибок на жестком диске во время установки
=================================================================
Версия программы Scandisk, выполняемая при установке второго выпуска 
Windows 98, только находит ошибки. Она не исправляет их. При наличии 
ошибок программа установки не сможет продолжать работу, пока они не 
будут устранены. Для устранения этих ошибок выйдите из программы 
установки и запустите программу ScanDisk для Windows 95 или MS-DOS. 
Ниже содержатся сведения об использовании программы ScanDisk для 
устранения этих ошибок.

Устранение ошибок на жестком диске:
Если в процессе установки появится сообщение о том, что для устранения 
ошибок на жестком диске следует запустить программу ScanDisk, 
выполните следующие шаги.

Если второй выпуск Windows 98 устанавливается поверх MS-DOS или 
предыдущей версии Windows, например, Windows 3.1:

1. Завершите работу Windows.

2. Если установка выполняется с дискет, вставьте в дисковод 
   установочный диск 1 и введите в командной строке:

	a:scandisk.exe /all

   где "a" - буква дисковода, в который вставлен диск с Windows.

3. Если установка выполняется с компакт-диска, вставьте компакт-диск 
   и введите следующую команду:

        d:\win98\scandisk.exe /all

   где "d" - буква дисковода, в который вставлен компакт-диск.

4. Следуйте инструкциям, появляющимся на экране, и исправляйте все 
   обнаруженные ошибки.

5. Войдите в Windows и снова запустите программу установки.

Если второй выпуск Windows 98 устанавливается поверх предыдущей версии 
Windows 98 или Windows 95:

1. Выйдите из программы установки.

2. Нажмите кнопку "Пуск", выберите в меню пункты "Программы", 
   "Стандартные", "Служебные программы" и запустите программу проверки 
   диска (ScanDisk).

3. Проверьте жесткие диски и все несущие диски на наличие ошибок и 
   устраните все найденные ошибки. Убедитесь, что проверены все диски, 
   иначе программа установки может продолжать обнаруживать ошибки.

Неполадки в работе программы ScanDisk:
Существует несколько случаев, в которых программа ScanDisk не может 
устранить ошибку или сама порождает ее.

При использовании программы сжатия диска DriveSpace 3:
Если на компьютере установлена программа сжатия диска Drvspace3, 
возможна нехватка основной памяти. Чтобы освободить память, попробуйте 
выполнить следующие действия:

* При работе в MS-DOS 6.x можно запустить программу Memmaker.exe для 
  освобождения объема памяти, достаточного для завершения программы 
  установки.
* Выполнение "чистой" перезагрузки с освобождением памяти описано в 
  разделе "Установка второго выпуска Windows 98 из MS-DOS".
* При работе в среде Windows 95 выполните проверку дисков.

Если при запуске программы ScanDisk из программы установки все еще не 
хватает памяти или возникают другие проблемы, запуск программы 
ScanDisk можно предотвратить, запустив программу установки с ключом 
/IS. Для этого введите команду:

	setup /is

ПРИМЕЧАНИЕ. Не рекомендуется пропускать запуск программы ScanDisk во 
время установки. Из-за этого возможно возникновение ошибок на жестком 
диске, которые повлекут неправильную установку или работу программного 
обеспечения второго выпуска Windows 98. 


7. Ошибки в CAB-файлах во время установки системы WINDOWS 98
============================================================
Во время установки Windows 98 или отдельного компонента, требующего 
копирования файлов с установочных дискет или компакт-диска Windows, 
могут выдаваться следующие сообщения:

 - Ошибка при распаковке: 
   "Не удается распаковать CAB-файл. Будет предпринята попытка 
   исправить ошибку. Нажмите кнопку ОК".

 - "Не удается скопировать файлы Windows 98. Не удается скопировать 
   файлы Windows 98. Протрите диск мягкой салфеткой, вставьте его 
   обратно и нажмите кнопку "ОК". Если сообщение появится снова, 
   обратитесь к разделу "Ошибки CAB-файлов" файла setup.txt. Он 
   находится в папке Win98 диска Windows 98."

Основных причин может быть несколько:

 - Повреждение компакт-диска Windows 98, грязь и царапины на его 
   поверхности.

 - Неисправность устройства чтения компакт-дисков. Высокий уровень 
   вибрации устройства чтения компакт-дисков, делающий невозможным 
   нормальное считывание данных.

 - Установлена недопустимо высокая тактовая частота процессора. 
   Распаковка архивных файлов Windows 98 требует интенсивной работы с 
   оперативной памятью. Если тактовая частота выходит за допустимые 
   пределы, извлечение данных может происходить с ошибками. Эти ошибки 
   могут быть вызваны также неисправностью системы охлаждения и 
   перегревом процессора.

 - Неисправность микросхем памяти или неправильная их установка. 
   Примером неправильной установки может служить совместное 
   использование микросхем памяти EDO с микросхемами другого типа или 
   применение микросхем с различным быстродействием. Нарушений в 
   работе Windows может при этом не наблюдаться, однако использование 
   дополнительных ресурсов и обращение к диску могут привести к 
   упомянутым ошибкам.

 - В BIOS и на вкладке "Устройства" был включен режим управления 
   шиной или Ultra DMA. Скорость передачи данных при этом может 
   оказаться слишком высока.

 - Использование программ управления памятью сторонней разработки.

 - Компьютер заражен вирусом.

Для устранения указанных неполадок выполните описанные ниже действия. 

1. Удалите компакт-диск из устройства чтения, поверните его на 
   четверть или пол-оборота, снова поместите в устройство чтения 
   дисков, нажмите кнопку OK.

2. Удалите компакт-диск из устройства чтения. Протрите его мягкой 
   салфеткой и снова поместите в устройство чтения. Нажмите кнопку OK.

3. Воспользуйтесь драйверами устройств чтения компакт-дисков для 
   реального режима. Если не удается найти драйверы для конкретной 
   модели устройства, попробуйте запустить драйверы с установочного 
   диска Windows 98. Записанные на этом диске служебные программы 
   обеспечивают поддержку большинства типов устройств чтения 
   компакт-дисков, включая (IDE) и (SCSI). Запустите установку 
   Windows из MS-DOS.

4. На жестком диске создайте папку "W98flat". Скопируйте в нее 
   содержимое папки Win98 с компакт-диска. Если это сделать не 
   удается, возможно, компакт-диск поврежден.

5. С помощью специальных программных средств проверьте компьютер на 
   наличие вирусов.

6. Запустите установку Windows 98. Воспользуйтесь для этого командой 

   "setup /c" (кавычки вводить не следует). Данный параметр отключает 
   запуск программы SMARTDrive. Скорость выполнения программы 
   установки несколько замедлится, но возможность сбоев при этом 
   также уменьшится.

9. Если ошибок распаковки CAB-файлов в Windows 98 перечисленными выше 
   способами избежать не удается, попробуйте вручную распаковать 
   файлы Windows 98 на жесткий диск, а затем запустить установку уже с 
   жесткого диска. Это потребует около 300 Мбайт свободной дисковой 
   памяти. Для распаковки файлов системы можно воспользоваться 
   служебной программой Ext.exe, которая находится на загрузочном 
   диске Windows 98, а также в папке \Oldmsdos компакт-диска 
   Windows 98. Чтобы распаковать файлы Windows 98 вручную, выполните 
   следующее:

   а. Поместите загрузочный диск Windows 98 в дисковод и перезагрузите 
      компьютер.

      ПРИМЕЧАНИЕ. Если загрузочный диск Windows 98 отсутствует, 
      обратитесь к разделу "Советы по установке драйверов устройств 
      чтения компакт-дисков реального режима" в части "Установка 
      второго выпуска Windows 98 из MS-DOS".

   б. Введите в командной строке "ext" (без кавычек).

   в. На соответствующий запрос введите путь к CAB-файлам, 
      расположенным в созданной ранее (шаг 4) папке W98Flat.

   г. В поле имени файла укажите "*.*".

   д. Введите путь к папке W98Flat для распакованных файлов. 

      *Примечание*. Файлы Precopy1.cab и Precopy2.cab распакованы 
      не будут.

   е. После того как все файлы будут распакованы, запустите программу 
      установки из папки W98Flat на жестком диске.

5. Если описанные выше меры не привели к положительному результату, 
   попробуйте снизить быстродействие компьютера. Этого можно добиться 
   следующим образом.

    - Измените параметры CMOS. На быстродействие компьютера влияют, 
      например, параметры управления шиной, внешнего/внутреннего 
      кэширования, параметры и быстродействие ОЗУ и пр. Сведения об 
      изменении этих настроек можно найти в документации на компьютер.


8. Удаление второго выпуска Windows 98 из системы
==================================================
Сохранение системных файлов:
Программа установки второго выпуска Windows 98 предоставляет 
пользователям возможность резервного копирования предыдущей 
установленной версии Windows на тот случай, если позже эту 
операционную систему потребуется удалить. Для включения этой 
возможности при установке следует установить флажок 
"Сохранить системные файлы". Программа установки создаст следующие 
скрытые файлы, необходимые для удаления Windows 98:
  * Winundo.dat
  * Winundo.ini
  * Winlfn.ini

ПРИМЕЧАНИЕ. Отсутствие этих файлов сделает удаление второго выпуска 
Windows 98 невозможным.

Если выполняется одно из следующих условий, установку второго выпуска 
Windows 98 нельзя будет отменить и программа установки не предложит 
сохранить имеющиеся системные файлы:

* Текущая версия Windows установлена на сжатом диске.
* Установка выполняется в другой каталог или не как обновление 
  предыдущей версии.
* Используется версия MS-DOS меньше 5.0.

ПРИМЕЧАНИЕ. Файлы, необходимые для удаления второго выпуска Windows 98, 
должны располагаться на локальном жестком диске. Их нельзя сохранить на 
сетевом или гибком диске. Если в наличии имеется два или более дисков 
с достаточным объемом свободного места, можно выбрать диск для 
сохранении данных об отмене установки.

После завершения установки Windows 98 некоторые действия могут сделать 
невозможной ее отмену. Ниже приведен список действий, удаляющих с 
жесткого диска данные об отмене установки.

* Преобразование жесткого диска к системе FAT32
* Сжатие жесткого диска программой DriveSpace

ПРИМЕЧАНИЕ. Перед установкой эти служебные программы выдадут 
предупреждение о потере данных, необходимых для отмены установки, в 
результате преобразования или сжатия.

Удаление второго выпуска Windows 98 с компьютера
Чтобы удалить второй выпуск Windows 98 и полностью восстановить 
предыдущую версию MS-DOS, Windows 3.x или Windows 95:

1. Нажмите кнопку "Пуск" и из меню "Настройка" выберите команду 
   "Панель управления".
2. Дважды щелкните значок "Установка и удаление программ".
3. На вкладке "Установка/удаление" нажмите кнопку 
   "Удалить Windows 98", а затем кнопку "Добавить/удалить".

Или, при наличии трудностей с запуском Windows 98, воспользуйтесь 
системным диском для загрузки компьютера, а затем запустите с него 
программу UNINSTAL.

ПРИМЕЧАНИЕ. Программа UNINSTAL требует завершения работы Windows 98. 
Если этого сделать не удается, перезапустите компьютер и после 
появления сообщения "Starting Windows 98" нажмите клавишу F8. Выберите 
режим загрузки "Command Prompt Only" и в командной строке введите 
UNINSTAL.

Если установка была завершена неудачно, и требуется восстановить 
предыдущую версию MS-DOS и Windows 3.x или Windows 95, можно запустить 
программу UNINSTAL из каталога \Windows\Command на жестком диске или 
на системном диске.

Если файлы отмены установки были сохранены не на диске C, добавьте 
параметр /w и укажите диск, на котором содержатся файлы. Например:

        uninstal /w e:

где e: является диском, содержащим предыдущие системные файлы.

Если же Windows 98 нормально работает, файлы, необходимые для отмены 
установки, можно удалить и тем самым освободить место на диске, 
выполнив следующие действия:

1. Нажмите кнопку "Пуск" и из меню "Настройка" выберите команду 
   "Панель управления".
2. Дважды щелкните значок "Установка и удаление программ".
3. На вкладке "Установка/удаление" выберите пункт "Старые системные 
   файлы Windows 3.x/MS-DOS" и нажмите кнопку "Удалить". Или нажмите 
   кнопку "Удалить системные файлы Windows 95".

После этого отменить установку Windows 98 будет нельзя.


9. Возможные неполадки при наличии сжатого диска
================================================
При наличии сжатого жесткого диска может возникнуть сообщение о 
нехватке свободного места на несущем диске сжатого диска. Программа 
установки должна скопировать некоторый файлы на загрузочный диск, на 
несущий диск загрузочного диска или на несущий диск диска, на котором 
расположен каталог Windows. При возникновении такого сообщения 
освободите немного места на указанном диске и снова запустите 
программу установки. Воспользуйтесь одним из следующих методов:

* Если возможно, установите Windows на несжатый диск.

* Удалите все ненужные файлы на несущем диске.

* При работе в Windows 3.1 и использовании постоянного файла подкачки 
  уменьшите его размер. На панели управления дважды щелкните значок 
  "386 Расширенный" и нажмите кнопку "Виртуальная память". Уменьшите 
  размер файла подкачки.

* Используйте программное обеспечение сжатия дисков для освобождения 
  места на несущем диске. Если используется программа DriveSpace или 
  DoubleSpace, выполните следующие действия:

  1. Завершите работу Windows.
  2. Запустите Drvspace.exe или Dblspace.exe.
  3. Выберите сжатый диск, на несущем диске которого требуется 
     освободить место.
  4. В меню "Диск" выберите команду "Изменить размер" и измените 
     объем свободного места, если требуется.

  Если используется программа DriveSpace Windows 95 или Drvspace3 из 
  пакета Plus!, выполните следующие действия:

  1. Запустите Windows
  2. Нажмите кнопку "Пуск " и запустите программу Drivespace из меню 
     "Программы" -> "Стандартные" -> "Служебные программы".
  3. Выберите сжатый диск, на несущем диске которого требуется 
     освободить место.
  4. В меню "Диск" выберите команду "Изменить размер" и измените 
     объем свободного места, если требуется.

Если используется другое программное обеспечение, такое как Stacker, 
обратитесь к его документации.

ПРИМЕЧАНИЕ. Возможно различие между объемом свободного места, 
требуемым программой установки, и реальным объемом свободного места на 
несущем диске. Некоторый объем диска занимает файл подкачки Windows. 
Если Windows не работает, это место может использоваться в других 
целях.

ПРИМЕЧАНИЕ. Если в процессе установки создается системный диск, 
проверьте чтобы он не был сжат.


10. Установка второго выпуска Windows 98 совместно с 
    операционной системой Windows NT
=====================================================
Второй выпуск Windows 98 нельзя установить поверх любой версии 
Windows NT, но обе эти системы могут сосуществовать на одном 
компьютере. Тем не менее, в целях обеспечения совместимости 
рекомендуется устанавливать каждую операционную систему на отдельный 
жесткий диск или раздел. Если Windows NT уже установлена, программа 
установки добавит Windows 98 в меню загрузки Windows NT для 
обеспечения выбора варианта загрузки между Windows 98 и Windows NT.

Если Windows NT перестала загружаться, загрузите компьютер с дисков 
аварийного восстановления Windows NT и выберите режим восстановления 
загрузочных файлов Windows NT.

При установке Windows 98 на компьютер с дисками, созданными 
Windows NT, возможно возникновение следующей ошибки:

"Жесткий диск имеет раздел FAT с размером кластера 64 КБ. Поскольку 
ScanDisk не может работать с такими дисками, установка не может быть 
продолжена. Для завершения установки необходимо произвести разделение 
жесткого диска и произвести форматирование с размером кластера в 32 КБ 
или меньше, а после этого повторить установку."

При запуске программы установки с параметром "/is" (т.е., Setup /is) 
программа ScanDisk не будет запущена и данная ошибка не возникнет.

Установка совместно с операционной системой Windows NT
------------------------------------------------------
Для установки двойной загрузки на компьютер класса x86 установите 
операционную систему обычным способом и измените файл Boot.ini, как 
показано ниже. Файл Boot.ini, автоматически создаваемый в корневом 
каталоге жесткого диска, содержит все сведения о запуске системы.

>>>Чтобы изменить файл Boot.ini:

1.  В проводнике Windows выберите команду "Свойства папки" в меню 
    "Вид" и установите флажок "Показать все файлы".

2.  Убедитесь, что флажок "Не показывать расширения для 
    зарегистрированных типов файлов" снят, и нажмите кнопку "OK".

3.  Щелкните правой кнопкой мыши файл Boot.ini и выберите пункт 
    "Свойства".

4.  Снимите флажок "Только чтение" и нажмите кнопку "OK".

5.  Правой кнопкой мыши щелкните файл Boot.ini, выберите команду 
    "Копировать", правой кнопкой мыши щелкните свободную область 
    обозревателя и выберите команду "Вставить". Будет создана 
    резервная копия файла с названием "Копия Boot.ini".

6.  Дважды щелкните значок файла Boot.ini.

7.  В разделе файла [operating systems] вставьте название и 
    расположение другой операционной системы, как показано в примере:

	[operating systems]
	C:\Winnt="Windows NT 4.0"
	C:\="Microsoft Windows"

8.  Сохраните и закройте файл Boot.ini.

9.  Щелкните правой кнопкой мыши файл Boot.ini и выберите пункт 
   "Свойства".

10. Установите флажок "Только чтение" и нажмите кнопку "OK".


11. Сообщения об ошибках установки
==================================
В этом разделе приведены конкретные сообщения, которые могут возникать 
с процессе установки, и рекомендации по дальнейшим действиям.

Сообщение SU0018
"Программе установки не удалось создать файлы на диске и произвести 
запуск Windows 98. Возможные причины: слишком большое количество 
файлов в корневой папке или переопределение имени диска сетевым 
программным обеспечением либо программой уплотнения диска."

Корневой каталог диска рассчитан на 512 или менее записей (файлов или 
папок). Это сообщение означает, что в корневом каталоге обнаружено 
максимально возможное число каталогов и программа установки не может 
создать файлы, требуемые для установки Windows 98. Переместите или 
удалите некоторые файлы из корневого каталога диска и снова запустите 
программу установки.

Сообщение "Неустранимая ошибка в процессе установки"
"Неустранимая ошибка в процессе установки: Продолжение установки в 
данной конфигурации системы невозможно. Выйдите из программы 
установки". Эта ошибка может быть вызвана разными обстоятельствами. 
Дополнительные сведения можно найти в разделах "Общие вопросы 
установки" и "Установка второго выпуска Windows 98 из MS-DOS".

Сообщения об ошибках в длинных именах файлов
Если появится сообщение "В этой папке обнаружено установленное 
приложение Long File Names. Продолжение установки невозможно", 
выйдите из программы установки и удалите Long File Names с компьютера 
при помощи команды Uninstall. Более подробные сведения можно найти 
в разделе "View Software".

Сообщения о нехватке памяти
Если возникает сообщение о нехватке памяти, объем основной памяти 
можно увеличить, вставив комментарии в начала строк загрузки 
резидентных программ и загружая драйверы в верхнюю память. Описание 
этих действий можно найти в базе знаний The Microsoft Knowledge Base 
(статья Q134399, "How to Increase Conventional Memory 
for MS-DOS-Based Programs") по адресу 
http://support.microsoft.com/support/kb/articles/q134/3/99.asp

Сообщения о нехватке места на диске
Освободить место на диске можно одним из следующих способов:

* Правой кнопкой мыши щелкните значок корзины и выберите команду 
  "Очистить корзину".

* Удалите файлы из папки кэша обозревателя Интернета.

* Удалите файлы с расширениями .bak и .tmp.

* Удалите папки с неиспользуемыми программами (предварительно создав 
  резервные копии файлов данных).

* Удалите старую папку MS-DOS, если не требуется наличие в компьютере 
  и Windows 98, и MS-DOS. (Перед этим проверьте наличие системного 
  диска, предоставляющего доступ к устройству чтения компакт-дисков.)

* Удалите скрытый файл Winundo.dat, оставшийся от предыдущей установки 
  Windows 95.

* Удалите старую папку Windows 3.1, если не требуется наличие в 
  компьютере и Windows 98, и Windows 3.1.

Не удается произвести запись в папку для временных файлов
Это сообщение может возникнуть в результате нехватки свободного места 
на диске, содержащем временную папку. Если необходимый объем 
присутствует на другом диске, измените расположение временного 
каталога следующей командой:

	Setup /T:<буква диска>:\TEMP

Если потребуется, освободите необходимый объем дисковой памяти и снова 
запустите программу установки. Файлы, которые можно удалить, 
перечислены в разделе "Сообщения о нехватке места на диске".

При наличии Multimedia Cloaking и установки Windows 95 с дискет 
возможен сбой при установке. При получении сообщения о невозможности 
прочесть файлы .cab выполните следующие действия:

1. Удалите строку со ссылкой на файл Cacheclk.exe из файлов 
   Config.sys и Autoexec.bat.
2. Перезагрузите компьютер.
3. Снова запустите программу установки.

Сообщения SU0010, SU0012, SU0015 и SU0016
Если во время установки выдается одно из указанных сообщений, 
обратитесь за подробностями к разделам "Установка второго выпуска 
Windows 98 совместно с операционной системой Windows NT" и 
"Установка Windows 98 совместно с операционной системой OS/2".

Сообщение SU0011
Если на жесткий диск установлена защита с паролем, установка не будет 
успешно завершена. Сначала следует удалить защиту с паролем. Более 
подробные сведения можно найти в документации по компьютеру.

Ошибка SU0013
Установка Windows 98 требует наличия загрузочного раздела MS-DOS. 
Если файловой системой загрузочного диска является HPFS или NTFS, 
перед выполнением установки необходимо создать загрузочный раздел 
MS-DOS. За дополнительными сведениями о создании загрузочного раздела 
MS-DOS обратитесь к документации по компьютеру.

Эта ошибка также может возникнуть при использовании программ 
управления разделами сторонних производителей типа EZ drive и 
Disk Manager. В этом случае следует перезагрузить компьютер и 
запустить программу установки из командной строки MS-DOS. 
Дополнительные сведения содержатся в разделе "Установка Windows 98 
из MS-DOS".

Сообщения в стандартном режиме
При получении одного из следующих сообщений удалите из файла 
Config.sys все диспетчеры памяти (такие как EMM386.exe, QEMM и 386Max) 
и снова запустите программу установки.

	Стандартный режим: Invalid DPMI return.
	Стандартный режим: Fault in MS-DOS Extender.
	Стандартный режим: Bad Fault in MS-DOS Extender.
	Стандартный режим: Unknown stack in fault dispatcher.
	Стандартный режим: Stack Overflow.

ПРИМЕЧАНИЕ. Если неполадки все еще возникают, вставьте обратно в файл 
Config.sys ссылку на EMM386.EXE и исключите все диапазоны адресов. 
Например:

	device=c:\windows\emm386.exe x=A000-FFFF

При получении этих сообщений или при зависании компьютера в процессе 
установки включите в SmartDrive двойную буферизацию. Некоторые диски 
SCSI и ESDI требуют двойную буферизацию.

Для включения двойной буферизации добавьте в начало файла Config.sys 
следующую строку:

	device=c:\windows\smartdrv.exe /double_buffer+

где "c:\windows" - каталог Windows.

Сообщение "Не удается создать резервные копии системных файлов"
Если это сообщение возникло при сохранении системных файлов, возможно, 
не хватает свободного дискового пространства, особенно на сжатых 
дисках. Освободите некоторое пространство на диске, куда требуется 
записать системные файлы (по умолчанию диск C), удалив ненужные файлы.

На большинстве компьютеров для возможности отмены установки требуется 
75 Мбайт. Если программа установки не предлагает сохранить системные 
файлы, возможно для этого не хватает места.

"Продолжение установки в данной конфигурации системы невозможно"
Такое сообщение возникает при наличии старого, несовместимого раздела 
диска. Перед началом установки создайте резервную копию файлов данных 
и заново создайте разделы на диске.

Сообщение SU0167
В компьютере уже существует файл или папка с именем "Desktop". 
Переименуйте или переместите этот объект и снова запустите программу 
установки.

Сообщение SU0167
На компьютере уже установлена операционная система, которая не может 
быть обновлена с помощью данной версии программы установки. Вам 
следует приобрести пакет обновления Windows 98.

Эта ошибка может выдаваться при попытке установки OEM-версии 
Windows 98 на прежнюю версию Windows.
-или-
Приложением была установлена OEM-версия библиотеки SetupX.dll, в связи 
с чем текущая версия Windows распознается как OEM-версия.

В последнем случае следует заменить файл SetupX.dll правильной 
версией с текущего установочного диска.

1. Поместите в дисковод компакт-диск Windows 95 или Windows 98.
2. Из командной строки перейдите к устройству чтения компакт-дисков.
3. Введите команду "Extract /a base4.cab SetupX.dll /l c:\"
4. Скопируйте с диска C: файл SetupX.dll в каталог Windows\System.
5. Запустите программу установки второго выпуска Windows 98 еще раз.

Сообщения SU99xxxx
SU99 является префиксом, который добавляется ко всем ошибкам, не 
имеющим четких определений. Обычно эти ошибки вызваны нехваткой нижней 
основной памяти. Если системный диск уже был создан, выйдите из 
программы установки, выключите компьютер, вставьте системный диск и 
загрузите компьютер. Запустите программу установки из командной строки 
MS-DOS.

"Недостаточно стандартной памяти для проверки дисков компьютера".

-или-

"Программе установки не удалось проверить жесткие диски компьютера".

Одна из этих ошибок возникает, когда программе установки не удается 
запустить программу ScanDisk для проверки жестких дисков компьютера. 
Это происходит из-за нехватки основной памяти или из-за того, что 
программа ScanDisk, запущенная программой установки, не может 
исправить обнаруженную ошибку. Чтобы обойти эти ошибки, перезагрузите 
компьютер в режиме MS-DOS и выполните команду ScanDisk /all. Затем 
снова запустите программу установки. Если опять возникнут ошибки, 
перезагрузите компьютер в режиме MS-DOS и выполните команду
Scandisk /all /surface. Это займет немного больше времени, но даст 
гарантию отсутствия ошибок на жестком диске. Если после запуска 
программы ScanDisk с этими параметрами работа программы установки не 
продолжится, запустите ее с параметром /is для предотвращения запуска 
программы ScanDisk.

ПРИМЕЧАНИЕ. Пропуск проверки дисков программой ScanDisk не 
рекомендуется и может выполняться только если диски точно не содержат 
ошибок. В противном случае возможны сбои в ходе установки и 
невозможность дальнейших загрузок компьютера.


12. Примечания об оборудовании
==============================
В этот раздел включены примечания о конкретных конфигурациях 
компьютеров, установка на которые второго выпуска Windows 98 может 
привести к нежелательным последствиям.

Sager NP8200 или Wedge Technologies 466/DX2
Внимание! При установке Windows 98 на переносной компьютер 
Sager NP8200 или Wedge Technologies 466/DX2 использование компьютера 
станет возможным лишь после установки предыдущей версии Windows.

Сетевые платы Plug and Play и 16-разрядные драйверы реального режима
При запуске 16-разрядных драйверов реального режима NIC (которые 
являются общими для DLC или других протоколов реального режима) 
плата Plug and Play может не работать, поскольку 16-разрядные драйверы 
NIC запускаются раньше, чем устанавливаются платы Plug and 
Play Windows 98. Кроме того, некоторые 16-разрядные драйверы NIC не 
распознают платы Plug and Play (большинство клонов NE2000 Plug and 
Play относятся к ним).

>>>Для использования плат Plug and Play с 16-разрядным драйвером NIC 
   необходимо:

1. Запустить программу Softset, поставляющуюся вместе с платой 
   Plug and Play, и отключить режим Plug and Play.

2. Дважды щелкнуть значок "Сеть" на панели управления и удалить 
   сетевую плату.

3. Дважды щелкнуть значок "Установка оборудования" на панели 
   управления - Windows 98 определит сетевую плату.

Если в дальнейшем драйвер NIC будет заменен на 32-разрядный 
защищенного режима, можно используя программу Softset вернуть режим 
Plug and Play для сетевой платы.

Компьютеры Intel EtherExpress 16 NICs и PCI
При использовании Intel EtherExpress 16 NIC в компьютерах PCI с 
некоторыми видеоадаптерами Diamond Speed Star PCI могут возникать 
проблемы с инициализацией и зависание системы. Это известная для 
Intel проблема не связана с Windows 98 и наблюдается в различных 
операционных системах. Поэтому в случае возникновения проблем с 
EtherExpress 16 в компьютерах PCI замените эти платы, перед тем как 
обращаться за помощью в Microsoft.

Обратитесь к вашим поставщикам за обновлением видео-BIOS для следующих 
видеоадаптеров:

- Diamond Speed Star PCI с BIOS версии 1.01
- Diamond Viper PCI VGA
- Diamond Stealth, Bios v1.03

IBM Thinkpad моделей 750x/755x/360x
При наличии одной из этих моделей (включая такие разновидности, как C, 
CE, CX, CS и CSE), найдите на доске объявлений IBM или форуме 
THINKPAD CompuServe обновленный BIOS и/или драйверы Mwave для 
компьютера перед установкой Windows 98.

Если BIOS не обновлен, могут возникнуть проблемы, если сдвинуть мышь 
при включении или отключении режима приостановки.

13. Диспетчеры памяти
=====================
Если на компьютере установлен диспетчер памяти QEMM, рекомендуется 
удалить его из файла Config.sys перед установкой Windows 98. Его 
можно добавить после установки Windows 98.

Следующие диспетчеры памяти несовместимы с Windows 98 либо могут 
привести к трудностям в работе системы. Рекомендуется удалить ссылки 
на них в файлах Config.sys и Autoexec.bat, воспользовавшись вместо 
этого совместимыми с Windows 98 программами Emm386.exe и Himem.sys 
для управления памятью.

        Allemm4.sys - All Charge 386 версии 3.1
        HPemm386.sys и HPemm486.sys
        HPmm.sys - диспетчер памяти HP (удалите его перед установкой)
        Диспетчер памяти Iemm.sys
        Intel(R) Expanded-Memory Emulator (Ilim386.sys)
        Диспетчер памяти Maximizer (Maximize.com)
        Диспетчер памяти NetRoom (RM386.sys)
        Диспетчер памяти QMAPS версии 5.16
        UMB PRO версии 1.07

ПРИМЕЧАНИЕ. В документации к 386MAX не рекомендуется устанавливать 
параметр EXT ниже 64. Программа установки не будет работать, если этот 
параметр равен 0.

ПРИМЕЧАНИЕ. При удалении строки, содержащей Rm386.sys из файла 
Config.sys, возможно придется установить Himem.sys, входящий в 
комплект Windows или MS-DOS. Это можно сделать, добавив в файл 
Config.sys строку:

        device=<путь>\himem.sys 

Здесь <путь> - место, где расположены файлы Windows или MS-DOS 
(например, C:\Windows).


14. Программы кэширования дисков
================================
Перед началом установки рекомендуется удалить программы кэширования 
диска сторонних поставщиков и заменить их программой Smartdrv.exe, 
которая поставляется вместе с Windows 98.

Следующие программы кэширования диска следует удалить самостоятельно, 
хотя программа установки в некоторых случаях может удалить их 
автоматически.

  386MAX (Qcache.exe) (нельзя использовать совместно со SmartDrive.)
  Cache.exe
  Cache.sys
  Flash (Flash.exe) (нельзя использовать совместно со SmartDrive.)
  Hyperdisk (нельзя использовать совместно со SmartDrive.)
  Icache.sys Disk-Cache utility
  IBMcache.sys Disk-Cache utility
  Mace Disk-Cache Utility (Mcache.sys)
  Norton Speed Drive (SPEEDRV)
  Norton Utilities версии 5.0 и 6.1 (DISKREET, NCACHE)
  PC-Cache (PC Tools) (следует удалить перед началом установки)
  PC-Kwik (Pc-kwik.exe)
  PC_Kwik Disk Accelerator
  PC-Kwik версии 1.59 (Pck.exe, Pskscrn.exe, Pckey.com)
  PC Tools(TM) (Pc-cache.com)
  Secretdisk II (Fast512.sys) (нельзя использовать 
      совместно со SmartDrive.)
  Super PC-Kwik (Superpck.exe)


15. Использование резидентных (TSR) программ
=============================================
APPEND (MS-DOS)
Не используйте служебную программу APPEND в среде Windows 98. 
Она мешает Windows и программам Windows создавать правильные имена 
полных путей (WFP) для используемых файлов.(WFP представляет собой 
полное имя файла, включающее букву диска и полный путь к файлу, 
начинающийся из корневого каталога.)

Autocon версии 2.0e, Bootcon версии 1.60 и Boot.sys 
Эти программы позволяют выбрать одну из нескольких конфигураций при 
загрузке компьютера. Так как программа установки Windows 98 изменяет 
только первый раздел в файлах Config.sys и Autoexec.bat, после 
завершения установки может потребоваться изменение этих файлов вручную. 
Более подробные сведения можно найти в документации по программе.

Cubit версии 3.01
При совместном использовании Cubit и Windows 98 не сжимайте папку 
Windows System и файлы, загружаемые из файла Config.sys.

Перед обновлением системы на второй выпуск Windows 98 убедитесь, что 
существующая версия Windows не сжата. После успешной установки второго 
выпуска Windows 98 файлы можно будет снова сжать.

Doubledisk версии 2.5
Возможны попытки Windows 98 получить доступ к "фиктивным" дискам, если 
Doubledisk использует их. Фиктивные диски появляются при создании 
сжатых дисков с помощью программы Doubledisk. Например, если имеются 
диски A, B и C, а с помощью программы Doubledisk создан диск F, 
Windows 98 и MS-DOS распознают диски D и E как правильные даже если 
они не существуют. Для отмены обращения Windows к фиктивным дискам 
воспользуйтесь служебной программой DRVOFF. Для получения этой 
программы обратитесь в Vertisoft.

Infinite Disk версии 2.1
Infinite Disk версии 2.1 не работает в среде Windows 98. Удалите 
программу Infinite Disk перед началом установки или обратитесь к 
дилеру за версией программы Infinite Disk, совместимой с Windows 98.

>>>Чтобы удалить программу Infinite Disk с компьютера:

1. Разожмите все сжатые файлы.

2. Удалите из файлов Config.sys и Autoexec.bat следующие строки:

	iddrv.sys
	idres.exe
	protect /c

3. Перезагрузите компьютер и снова запустите программу установки.

NetWare(R) NetBIOS
Во многих конфигурациях резидентный модуль NetWare NetBIOS несовместим 
с Windows 98. Как правило, программа установки обнаруживает 
NetWare NetBIOS и удаляет из компьютера команду его запуска. 
Тем не менее, если NetWare NetBIOS запускается не из файла 
Autoexec.bat (а, например, из другого пакетного файла), программа 
установки не сможет найти его. В таком случае ссылку на программу 
NetWare NetBIOS следует удалить вручную. В Windows 98 включен протокол 
связи, совместимый с IPX/SPX, который можно использовать вместо 
NetBIOS. Сведения о настройке Windows 98 для работы с ним содержатся 
в файле Networks.txt.

Newspace версии 1.07
Windows 98 несовместима с этой программой сжатия диска.

Norton Desktop для Windows (Ep.exe)
Если из файла Autoexec.bat загружается программа восстановления данных 
Erase Protect (Ep.exe), отключите ее на время установки Windows, 
особенно при обновлении предыдущей версии Windows. Так как программа 
Erase Protect пытается сохранить копии всех удаленных файлов, во время 
установки для этого может не хватить места на диске.

Запускать программу Erase Protect рекомендуется при наличии 
постоянного файла подкачки. При использовании временного файла 
подкачки снимите защиту с удаления файлов с расширениями .tmp и .swp.

Norton Disklock версии 3.01 и 3.5 не работает в среде Windows 98. 
Удалите эту программу перед началом установки или обратитесь в 
Symantec за версией программы Disklock, совместимой с Windows 98.

Сведения об удалении программы Disklock с компьютера можно найти в 
документации по этой программе.

PC Tools Deluxe(TM) версии 7.1 (DATAMON)
Если программы PC Tools DATAMON запущена с параметром \SENTRY+, 
программа установки может прервать работу с сообщением о нехватке 
места на диске. Не запускайте программу DATAMON с параметром \SENTRY+.

Не запускайте программу DATAMON после загрузки программы SmartDrive. 
Если программа DATAMON будет загружена после программы SmartDrive, 
может произойти сбой в работе компьютера. Сведения об удалении 
программы DATAMON можно найти в документации по PC Tools.

PC Tools Deluxe версии 6.0 (DESKTOP)
При попытке запустить PC Tools в среде Windows из командной строки 
компьютер может перезагрузиться или в его работе может возникнуть 
сбой. Эти же неполадки могут возникнуть при использовании команд 
MS-DOS в командной строке при работе Windows.

Эти проблемы устранены в PC Tools версии 7.0. Тем не менее, не 
загружайте программу DESKTOP из командной строки перед запуском 
Windows. В противном случае при завершении работы Windows может 
возникнуть сбой. Программу DESKTOP рекомендуется запускать из файла 
PIF, поставляемого с Windows 98.

SUBST (MS-DOS)
При выполнении установки могут возникнуть проблемы с программой SUBST. 
После завершения установки Windows 98 ее можно смело вернуть обратно.

UNDELETE
При запуске программы UNDELETE с параметром SENTRY, во время установки 
могут возникнуть неполадки, а все свободное дисковое пространство 
окажется занятым. Перед выполнением установки рекомендуется удалить 
ссылку на файл UNDELETE из файлов Config.sys и Autoexec.bat.

ПРИМЕЧАНИЕ. Параметр SENTRY очищает скрытый каталог с именем Sentry. 
При удалении файла программа UNDELETE перемещает его в каталог Sentry. 
Перед удалением ссылки на файл UNDELETE из файлов Config.sys и 
Autoexec.bat освободите дисковое пространство, выполнив на всех дисках 
команду UNDELETE/PURGE.

Если не убрать ссылку на программу Undelete, программа установки 
сделает это автоматически.
